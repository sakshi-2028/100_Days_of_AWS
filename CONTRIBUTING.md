# Contributing to 100 Days of AWS

Thanks for checking out this repo! While this is primarily my personal learning journal, suggestions and small fixes are always welcome.

## How you can help

- 🐛 **Found a typo or error?** Open an issue or a PR.
- 💡 **Got a better explanation?** Suggest it via PR — I'm always learning.
- 🔗 **Know a great resource for a topic?** Add it to the relevant `DayXX_*.md` under a "Resources" section.
- 📸 **Want to share your own AWS learning?** Fork the repo and start your own journey!

## File format

Each `DayXX_<Topic>.md` follows this template:

```markdown
# Day N: <Topic Name>

**Date:** YYYY-MM-DD

## What
## Why
## Steps (Console)
## Steps (CLI)
## Notes
```

Please keep new days consistent with this template.

## Reporting an issue

When opening an issue, include:

- **Day / topic** the issue is about
- **What's wrong** (typo, factual error, broken link, unclear step)
- **Your suggested fix** (optional but appreciated)

## Pull requests

- One topic per PR — keep changes focused.
- Match the existing tone (concise, beginner-friendly).
- If you add a new day, also update the table in `README.md`.

Thanks again — happy learning! ☁️
